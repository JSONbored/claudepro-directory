cat > CLAUDE md << 'EOF'
## Project: Enterprise Application Platform

## Tech Stack
- Frontend: React 18, TypeScript 5.2, Tailwind CSS
- Backend: Node js 20, PostgreSQL 15
- Infrastructure: AWS, Kubernetes

## Project Structure
- /frontend - React application
- /backend - API services  
- /shared - Common utilities
- /docs. - Architecture documentation

## Code Conventions
- TypeScript strict mode enabled
- 2-space indentation
- Functional components with hooks
- Test coverage minimum: 80%

## Forbidden Directories
- node_modules/
- dist/
- .git/
- coverage/
EOF`,
      tip: "Place CLAUDE md files strategically—root for project-wide, subdirectories for module-specific rules"
    },
    {
      title: "Step 3: Setup Multi-Directory. Access",
      description: "Configure Claude to access multiple repositories simultaneously",
      code: `# Add multiple directories to workspace
claude --add-dir ../backend --add-dir ../frontend --add-dir ../shared-libs

## Or dynamically during session
/add-dir ../documentation
/add-dir ~/company/design-system
/add-dir ../infrastructure`,
      tip: "Add directories in order of importance—Claude prioritizes earlier additions when context is limited"
    },
    {
title: "Step 4: Configure Git Worktrees",
      description: "Enable parallel development on multiple features",
      code: `# Create worktrees for parallel work
git worktree add ../project-feature-auth feature/authentication
git worktree add ../project-feature-api feature/api-redesign
git worktree add ../project-hotfix hotfix/critical-bug

## Launch separate Claude sessions
cd ../project-feature-auth && claude
## In new terminal:
cd ../project-feature-api && claude`,
      tip: "Each worktree maintains independent context—perfect for complex features requiring full attention"
    },
    {
title: "Step 5: Optimize Context Window",
      description: "Configure token usage for maximum efficiency",
      code: `# Create optimization config
cat > .claude/settings json << 'EOF'
{
  "contextWindow": {
    "maxTokens": 200000,
"reserveTokens": 40000,
    "fileFilters": {
      "exclude": ["*.test js", "*.spec ts", "*.min js"],
      "prioritize": ["src/core/*", "src/api/*"]
    }
  },
  "performance": {
    "lazyLoad": true,
    "cacheStrategy": "aggressive",
    "compressionLevel": "high"
  }
}
EOF'`,
      tip: "Reserve 20% of context window for Claude's responses—performance degrades when approaching limits"
    },
    {
      title: "Step 6: Create Custom Commands",

## Enterprise Integration Patterns

### Monorepo Tool Selection

Based on extensive research and production deployments:

**Nx** (Best for Complex Enterprise):
- Superior caching and dependency management
- Rich plugin ecosystem
- Steeper learning curve but maximum capability

**Turborepo** (Best for Rapid Adoption):
- Minimal configuration required
- Excellent Vercel ecosystem integration  
- Perfect for teams new to monorepos

**Lerna + Nx** (Migration Path):
- Maintain existing Lerna workflows
- Gain Nx performance benefits
-. Gradual transition strategy

### Security Configuration

<InfoBox type="warning" title="Enterprise Security Requirements">
Always implement these security measures in production:
- Audit logging for all Claude operations
- Restricted access to credentials and secrets
- Approval workflows for destructive operations
- Regular security scans on generated code
</InfoBox>
