description: >-
  Monitors and logs database query performance metrics with slow query
  detection, N+1 analysis, and optimization suggestions using PostgreSQL
  pg_stat_statements, Prisma query logging, Sequelize query logging, TypeORM
  query logging, and Bullet N+1 detection patterns. This PostToolUse hook
  automatically detects database query files and commands, analyzes query
  patterns for performance issues, identifies N+1 query problems, detects slow
  queries exceeding configurable thresholds, monitors database connection pools,
  provides query plan analysis and optimization hints, and logs performance
  metrics for historical analysis. Features include automatic slow query
  detection and alerting with configurable thresholds (default: 1000ms), N+1
  query pattern identification detecting loops with database queries and
  suggesting eager loading, query execution time tracking and statistics with
  detailed performance metrics, database connection pool monitoring tracking
  active connections and pool utilization, query plan analysis and optimization
  hints using EXPLAIN/ANALYZE for PostgreSQL, support for PostgreSQL 17+
  (pg_stat_statements), MySQL 8.0+, SQLite query logs, Prisma 5.x query logging,
  Sequelize 6.x/7.x query logging, and TypeORM 0.3.x query logging, SELECT *
  pattern detection warning about unnecessary data transfer, missing LIMIT
  clause detection preventing unbounded result sets, and unindexed WHERE clause
  detection suggesting index creation.
