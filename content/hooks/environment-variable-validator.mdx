  configuration across environments. This PostToolUse hook provides

  configuration across development, staging, and production environments.
  Features include automatic validation when .env files are modified with
  real-time validation feedback, required environment variable checking with
  cross-environment consistency validation, security validation for insecure
  defaults and weak secrets with minimum length requirements (16+ characters for
  secrets, 32+ for JWT/encryption keys), format validation for URLs, emails,
  ports, and booleans with comprehensive type checking, cross-environment
  consistency checking comparing .env, .env.local, .env.development,
  .env.staging, and .env.production files, production vs development environment
  validation with environment-specific security rules, configuration schema
  validation with Docker Compose environment variable validation, and
  comprehensive error reporting and security warnings with detailed validation
  summaries.
