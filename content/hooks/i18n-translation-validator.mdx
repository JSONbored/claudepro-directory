  different language files. This PostToolUse hook provides comprehensive i18n
  translation validation with comprehensive translation key validation across

  missing and orphaned translation key detection using jq for nested key
  extraction and comparison with base translation files, JSON structure and
  syntax validation for translation files with jq validation and error
  reporting, pluralization rule compliance checking with locale-specific
  pluralization validation, translation completeness percentage reporting with
  detailed completeness metrics and missing key counts, multi-format support
  (JSON, YAML, gettext PO files) with format-specific validation (jq for JSON,
  msgfmt for PO, basic validation for properties), variable placeholder
  validation and consistency checking with pattern matching for {{variable}},
  {variable}, %variable%, ${variable}, and printf-style placeholders, and
  character encoding and special character verification with non-ASCII character
  detection and locale-specific validation (RTL languages, CJK languages). The
  hook automatically detects translation files in common directories (locales/,
  i18n/, lang/, translations/), validates file format and syntax, compares with
  base translation files, detects missing and orphaned keys, validates variable
  placeholders, checks locale-specific requirements, and provides comprehensive
  validation reports with actionable recommendations.
